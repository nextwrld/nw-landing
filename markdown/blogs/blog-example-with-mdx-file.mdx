## Contexto

En LATAM, miles de gimnasios pequeños y medianos operan con una mezcla de papel, WhatsApp y planillas.

Eso genera un problema silencioso, pero caro: **no hay control real**.

- Socios vencidos que entran igual
- Cobros que se pierden o se registran mal
- Información duplicada o desactualizada
- “La operación vive en la cabeza” del dueño o la recepción

El objetivo del proyecto fue claro: **digitalizar la operación de manera inmediata**, sin exigir hardware ni procesos complejos.

---

## El problema real

El problema no es “falta de software”.

El problema es un sistema operativo informal:

- El gimnasio no puede saber en tiempo real quién está vigente y quién no
- El acceso se decide por memoria, mensajes o planillas
- El ingreso no queda registrado ni es auditable
- La adopción de tecnología se frena por fricción inicial (“no voy a cargar todo otra vez”)
- El dueño no tiene visibilidad financiera básica: ingresos, vencimientos y estado real de la base

La consecuencia: **pérdida directa de dinero y tiempo**, y una operación que no escala.

## MVP (Fase 0): control operativo inmediato

El MVP se diseñó para reemplazar por completo el control manual y las planillas.

### Módulos del MVP

**1) Gestión de socios**

- Alta / edición / baja
- Campos mínimos: nombre, cédula/DNI, contacto, plan, fechas, estado
- Búsqueda rápida por cédula o nombre
- Estado calculado dinámicamente (vigente / vencido / suspendido)

**2) Planes y membresías**

- CRUD de planes (duración, precio)
- Asignación al crear el socio
- Estado por fecha de fin (sin cálculos manuales)

**3) Control de acceso manual (recepción)**

- Check-in por cédula
- Validación instantánea: vigente / vencido / suspendido / no encontrado
- Registro completo de intentos (bitácora auditable)
- Resumen diario: entradas, vencimientos, alertas básicas

**4) Importación masiva CSV (anti-fricción)**

- Plantilla descargable
- Pre-validación y vista de revisión antes de confirmar
- Detección de duplicados / errores de formato
- Posibilidad de actualizar registros existentes
- Log de operación (quién, cuándo, cuántos)
- Mensajes claros: “50 cargados, 3 con errores”

**5) Roles**

- Administrador: configuración + gestión total
- Recepción: check-in + listado de socios

**6) Configuración**

- Días de gracia
- Zona horaria
- Cooldown entre accesos
- Parámetros globales del gimnasio

## Flujos clave (lo que el gimnasio hace todos los días)

| Flujo | Qué hace el usuario | Valor real |
| --- | --- | --- |
| Alta individual | registra un socio en recepción | operación diaria rápida |
| Carga masiva | sube CSV y digitaliza la base | onboarding en minutos |
| Check-in por cédula | valida acceso en segundos | orden + control |
| Auditoría diaria | revisa entradas y vencimientos | seguridad y control operativo |

---

## Criterios de éxito del MVP

El MVP se considera exitoso si:

1. El gimnasio digitaliza su base en **menos de 10 minutos** con CSV
2. La recepción hace check-in sin fricción ni errores
3. Se elimina el uso de Excel/WhatsApp para validar accesos
4. Se genera un historial auditable de accesos y vencimientos automáticamente

El resultado buscado no es “tener software”, sino **recuperar control**.

## Evolución del producto: de control a plataforma (roadmap)

Gym Access OS está diseñado como un sistema que crece por fases:

### Fase 1 — SaaS Core (multi-gimnasio)

- Multi-tenant (varios gimnasios)
- Panel central (administración global)
- Pagos y métodos (efectivo, transferencia, etc.)
- Dashboard financiero básico
- Suscripción mensual por gimnasio
- Backups y usuarios internos

**Resultado:** se consolida el modelo SaaS con cobro recurrente y operación escalable.

### Fase 2 — Expansión operativa

- Reservas de clases y cupos
- Asistencia por zona/clase
- Pagos online
- Reportes de retención y uso
- RBAC avanzado (roles finos)
- Registro de gastos y reportes exportables

**Resultado:** el sistema pasa de “controlar” a **administrar todo el negocio**.

### Fase 3 — Experiencia del socio

- Portal/app del socio
- QR/PIN digital
- Notificaciones de vencimientos y promociones
- Encuestas y feedback

**Resultado:** más retención, más fidelización, menos dependencia de apps externas.

### Fase 4 — Automatización e inteligencia

- Alertas de riesgo de abandono
- Recomendaciones de precios/promos
- KPIs y comparativas históricas
- Integraciones contables/marketing
- Automatización avanzada

**Resultado:** de sistema de gestión → **motor de decisiones**.

### Fase 5 — Ecosistema y expansión regional

- API pública
- Multi-sedes / franquicias
- Marketplace de extensiones
- Partners / distribuidores
- White-label

**Resultado:** plataforma regional adaptable a distintos tamaños de negocio.
